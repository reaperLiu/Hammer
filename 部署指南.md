# GitHub 部署指南

## 🚀 快速部署到GitHub Pages

### 第一步：创建GitHub仓库

1. **登录GitHub**
   - 访问 [github.com](https://github.com)
   - 使用你的账号登录

2. **创建新仓库**
   - 点击右上角的 "+" 号
   - 选择 "New repository"
   - 仓库名称建议：`chinaidcard` 或 `id-card-validator`
   - 设置为 Public（公开）
   - 不要勾选 "Initialize this repository with a README"
   - 点击 "Create repository"

### 第二步：本地Git初始化

在项目目录中运行以下命令：

```bash
# 初始化Git仓库
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: 中国身份证验证器"

# 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/chinaidcard.git

# 推送到GitHub
git push -u origin main
```

### 第三步：启用GitHub Pages

1. **进入仓库设置**
   - 在你的GitHub仓库页面，点击 "Settings" 标签
   
2. **配置GitHub Pages**
   - 在左侧菜单找到 "Pages"
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "main"
   - Folder 选择 "/ (root)"
   - 点击 "Save"

3. **等待部署**
   - 几分钟后，你的网站就会在以下地址可用：
   - `https://你的用户名.github.io/chinaidcard/`

## 🌐 访问你的网站

部署成功后，你可以通过以下方式访问：

- **主页**: `https://你的用户名.github.io/仓库名/`
- **验证器**: `https://你的用户名.github.io/仓库名/standalone_ui.html`

## 📝 更新网站内容

当你需要更新网站内容时：

```bash
# 添加更改的文件
git add .

# 提交更改
git commit -m "更新描述"

# 推送到GitHub
git push
```

GitHub Pages会自动重新部署你的网站。

## ⚙️ 自定义域名（可选）

如果你有自己的域名：

1. 在仓库根目录创建 `CNAME` 文件
2. 文件内容写入你的域名，如：`idcard.yourdomain.com`
3. 在你的域名DNS设置中添加CNAME记录指向 `你的用户名.github.io`

## 🔒 安全注意事项

1. **不要提交敏感信息**：如API密钥、密码等
2. **检查.gitignore**：确保不必要的文件不被提交
3. **定期更新**：保持代码和文档的更新

## 🆘 常见问题

### Q: GitHub Pages没有更新？
A: 等待几分钟，GitHub Pages部署需要时间。也可以在仓库的Actions标签查看部署状态。

### Q: 网页显示404错误？
A: 检查文件名是否正确，确保`index.html`在根目录。

### Q: 如何删除仓库？
A: 在仓库设置页面最底部有"Delete this repository"选项。

### Q: 如何让仓库私有？
A: 在仓库设置中可以改变仓库的可见性，但GitHub Pages需要付费账户才能在私有仓库中使用。
